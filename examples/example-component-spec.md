# Example Component Spec: Executive Proof Card

This example shows the expected level of component guidance: enough structure for a designer, engineer, or AI agent to produce consistent branded output.

## Purpose

Use this card to highlight a proof point, operating principle, or system capability.

## Anatomy

- eyebrow label
- title
- short supporting copy
- optional metric or proof point
- optional action link

## Token usage

| Element | Recommended token |
|---|---|
| Background | `--color-brand-slate` |
| Border | `--color-border-default` |
| Eyebrow | `--color-brand-amber` |
| Body text | `--color-text-muted` |
| Radius | `--radius-lg` |
| Padding | `--space-lg` |

## Content guidance

Use short, direct copy. This component should clarify one idea, not summarize an entire case study.

## Example copy

**Eyebrow:** Operating system

**Title:** Stabilize the system before scaling spend

**Body:** A growth system performs better when measurement, ownership, taxonomy, and decision rhythm are clear before budget increases.
