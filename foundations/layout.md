# Layout Foundation

Layout rules describe how the starter system organizes space, hierarchy, and responsive behavior.

## Principles

- Use clear section boundaries.
- Keep page structure scannable before decorative.
- Pair important claims with supporting context or evidence.
- Prefer reusable layout patterns over one-off page decisions.
- Make handoff expectations explicit for designers, engineers, and AI tools.

## Container guidance

| Pattern | Guidance |
|---|---|
| Page container | Use a max-width container for readable content areas. |
| Section spacing | Use larger spacing between major sections than within components. |
| Card grids | Use equal-height cards only when content density is comparable. |
| Hero layout | Pair a primary message with one clear action and optional supporting proof. |
| Form layout | Keep fields grouped, labeled, and supported with helper copy. |

## Responsive behavior

- Collapse multi-column hero layouts into a single column on small screens.
- Preserve CTA order when stacking content.
- Keep touch targets large enough for mobile use.
- Avoid hiding essential proof or context on smaller screens.

## Review checklist

- Does the page hierarchy make sense without visual styling?
- Does the primary action appear before secondary actions?
- Are sections grouped by decision or user need?
- Are responsive breakpoints intentional rather than accidental?
- Are layout rules documented clearly enough for front-end handoff?
