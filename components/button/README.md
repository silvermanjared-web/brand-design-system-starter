# Button Component

Buttons should be direct, readable, and action-oriented.

## Variants

| Variant | Use |
|---|---|
| Primary | Main page action |
| Secondary | Supporting action |
| Ghost | Low-emphasis action |
| Text | Inline or low-friction navigation |

## States

- default
- hover
- focus
- disabled
- loading, if implementation supports it

## Guidance

Button labels should be specific. Prefer `View roadmap`, `Download brief`, or `Start audit` over vague labels like `Learn more` when context allows.
