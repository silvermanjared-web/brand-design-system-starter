# Claude Handoff

Use this repo as a portable brand design system starter.

## Your job

When reviewing or generating work from this repo:

1. Read `README.md` first.
2. Read `design-tokens/tokens.json`.
3. Read the relevant foundation files.
4. Read only the component files needed for the task.
5. Preserve the brand posture: clear, senior, structured, calm, and modern.
6. Do not invent assets, fonts, or proprietary claims.
7. Prefer semantic tokens over hardcoded values.
8. Flag gaps instead of guessing.

## Output expectations

When producing design or front-end recommendations, include:

- what changed
- which tokens were used
- which component conventions were followed
- any accessibility risks
- any missing brand inputs
- what a human should review before shipping

## Do not

- expose private client material
- assume paid font rights
- create fake brand assets
- overcomplicate the system
- generate ornate UI that conflicts with the brand posture
